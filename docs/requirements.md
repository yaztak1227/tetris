# Tetris Requirements

## 目的

Mac ネイティブアプリとして起動できる、現代的なテトリス系ゲームを作成する。基本操作に加えて、Tetris Online Poland / Tetris Online Japan 系で期待される対戦・スコアリング・練習機能を取り込み、T-Spin を戦略として使えるゲーム体験を目指す。

## 参考情報

- Tetris Online Poland 公式サイト: https://tetrisonline.pl/
  - 「友人や世界中のプレイヤーと遊べるオンライン Tetris」と説明されている。
- Hard Drop Wiki: Tetris Online (Japan): https://harddrop.com/wiki/Tetris_Online_%28Japan%29
  - TOP は TOJ の非公式サーバー/クライアントとして言及されている。
  - TOJ には Marathon、Time Attack、Challenge、VS COM、Multi などのモードがある。
- Tetris Wiki: Tetris Online (Japan): https://tetris.wiki/Tetris_Online_%28Japan%29
  - 現在はファンメイドクライアント Tetris Online Poland で TOJ を遊べること、Premium 機能やアイテムが利用可能であることが記載されている。
- Tetris 一般仕様: https://en.wikipedia.org/wiki/Tetris
  - Next キュー、ソフトドロップ、ハードドロップ、ホールドなどが一般的な機能として説明されている。

## スコープ

### 対象

- 1人用 Tetris プレイ
- TOP/TOJ を参考にしたゲームモード
- T-Spin 判定と T-Spin を含むスコアリング
- 将来のオンライン対戦を見据えた攻撃量/ガベージ仕様
- キー操作、ホールド、Next キュー、ランキング向け記録
- macOS 上で通常のアプリケーションとして起動できる配布形態

### 対象外

- 初期リリースでの実オンラインマッチング
- 課金、アイテム購入、アカウント課金管理
- Tetris ブランド公式アセットの利用
- TOP/TOJ クライアント互換プロトコルの実装
- 初期リリースでの Windows / Linux / モバイル向けネイティブ配布

## ゲームフィールド

- フィールドは幅 10 マス、高さ 20 マスを基本表示領域とする。
- 上部にスポーン用の非表示行を持てること。
- ブロックがスポーン位置に置けない場合はゲームオーバーとする。
- ラインが横一列埋まった場合、そのラインを消去し、上のブロックを下へ落とす。
- 消去ライン数に応じてレベルを上げ、レベル上昇に応じて自然落下速度を段階的に速くする。
- ミノが地面または固定ブロックに接地しても即時固定せず、一定時間のロック遅延を設ける。
- 接地中に左右移動または回転が成功した場合、一定回数までロック遅延をリセットできる。
- ロック遅延リセットの上限に達した場合、またはロック遅延時間を消費した場合はミノを固定する。

## テトリミノ

- I, O, T, S, Z, J, L の 7 種類を使用する。
- ランダマイザは 7-bag 方式を採用する。
- Next キューは最低 5 個、設定により 6 個以上表示可能にする。
- Hold は 1 手につき 1 回だけ使用可能にする。
- Hold 未使用時は現在ミノを Hold に格納し、次のミノをスポーンする。
- Hold 使用済み時は、現在ミノと Hold ミノを入れ替える。

## 操作

- 左右移動
- ソフトドロップ
- ハードドロップ
- 右回転

- 180 度回転

- 一時停止
- リスタート

## 回転仕様

- 回転システムは SRS 相当を採用する。
- I, O, T, S, Z, J, L それぞれに回転状態を持つ。
- 壁や床、既存ブロックに衝突する回転は、キックテーブルを順に試す。
- いずれのキックでも成立しない場合、回転は失敗する。
- T-Spin 判定のため、最後に成功した操作が T ミノの回転だったかを保持する。
- 接地中でも回転キックが成立する場合は移動/回転を許可し、T-Spin や壁際回転が成立できる操作感にする。

## T-Spin 要件

T-Spin は必須機能とする。

### 判定条件

- 現在ミノが T ミノである。
- 最後に成功した移動系アクションが回転である。
- T ミノがロックされた時点で、T ミノ中心の四隅のうち 3 箇所以上が以下のいずれかで埋まっている。
  - フィールド外
  - 固定済みブロック
- 上記条件を満たしたうえでライン消去が発生した場合、消去ライン数に応じて T-Spin Single / Double / Triple として扱う。
- ライン消去が発生しない場合は T-Spin No Line として扱う。

### Mini 判定

- 初期実装では T-Spin Mini は任意対応とする。
- Mini を対応する場合は、T ミノ正面側の角、使用キック、消去ライン数をもとに通常 T-Spin と区別する。
- Mini 判定を実装しない間は、3-corner 条件を満たすものを通常 T-Spin として扱う。

### 表示

- T-Spin 成功時は画面上に以下のイベント名を表示する。
  - T-SPIN
  - T-SPIN SINGLE
  - T-SPIN DOUBLE
  - T-SPIN TRIPLE
- Back-to-Back 継続時は B2B 表示を追加する。

## スコアリング

- ライン消去、ソフトドロップ、ハードドロップ、T-Spin、Back-to-Back、コンボをスコア対象にする。
- Tetris、T-Spin Single/Double/Triple は Back-to-Back 対象にする。
- Back-to-Back 対象の手が連続した場合、ボーナスを付与する。
- 1 ライン以上の消去が連続した場合、コンボ数を増やす。
- ライン消去がないロックではコンボをリセットする。

推奨スコア:

| アクション | 基本点 |
| --- | ---: |
| Single | 100 |
| Double | 300 |
| Triple | 500 |
| Tetris | 800 |
| T-Spin No Line | 400 |
| T-Spin Single | 800 |
| T-Spin Double | 1200 |
| T-Spin Triple | 1600 |
| Soft Drop | 1 / cell |
| Hard Drop | 2 / cell |

## 攻撃量/ガベージ要件

将来の Multi / VS COM 実装に備え、ライン消去イベントから攻撃量を計算できること。

推奨攻撃量:

| アクション | 攻撃量 |
| --- | ---: |
| Single | 0 |
| Double | 1 |
| Triple | 2 |
| Tetris | 4 |
| T-Spin Single | 2 |
| T-Spin Double | 4 |
| T-Spin Triple | 6 |
| Back-to-Back bonus | +1 |
| Combo bonus | コンボ数に応じて加算 |

- 受信ガベージはキューとして保持する。
- ライン攻撃が発生した場合、まず受信ガベージを相殺する。
- 相殺後に残った攻撃量を相手へ送る。
- 受信ガベージの穴位置は一定確率で変化する。

## ゲームモード

TOP/TOJ を参考に、以下のモードを段階的に対応する。

### Marathon

- 標準的なスコアアタック。
- レベル上昇に応じて落下速度を上げる。
- 一定ライン数またはゲームオーバーまでプレイする。

### Time Attack / 40 Lines

- 40 ライン消去までのタイムを競う。
- ランキング記録対象にする。

### Challenge / Score Attack

- 制限時間内のスコアを競う。
- 10 分モードを基本候補とする。

### VS COM

- AI 相手にガベージ攻撃ありで対戦する。
- AI の強さを複数段階から選べる。

### Multi

- 将来的にオンライン対戦に対応する。
- ルーム作成、参加、観戦、チャット、リザルト表示を想定する。

### Practice / Training

- ミノ順固定、T-Spin 練習、無限リスタート、速度設定を提供する。
- T-Spin セットアップ練習用に盤面プリセットを読み込めること。

## ランキング/記録

- モード別にベストスコア、ベストタイム、最大コンボ、最大 Back-to-Back、T-Spin 回数を保存する。
- 初期実装ではローカル保存でよい。
- 将来的にはオンラインランキングへ拡張できる設計にする。

## 非機能要件

- キーボード入力からゲーム状態への反映までの遅延を最小化し、操作応答性を重視する。
- 左右移動、回転、ハードドロップ、Hold は入力直後のフレームで反映されることを目標にする。
- DAS/ARR/SDF の設定により、上級者向けの高速入力でも意図した操作が抜けにくいこと。
- Lock Delay、DAS、ARR、SDF、Next 表示数は設定値として調整できる設計にする。
- 長時間プレイ中も入力処理が描画負荷やエフェクトに阻害されないこと。
- 60 FPS 相当で描画できること。
- モバイルよりもキーボード操作のデスクトップ体験を優先する。
- macOS の `.app` として Finder、Dock、Launchpad から起動できること。
- アプリ起動後、ネットワーク接続なしで 1 人用モードを開始できること。
- macOS 標準のウィンドウ操作、フルスクリーン、アプリ終了に対応すること。
- ゲームロジックは描画処理と分離し、テスト可能にする。
- T-Spin 判定、ライン消去、ホールド、7-bag、スコアリングは単体テスト対象にする。

## 受け入れ基準

- 新規ゲームを開始すると 10x20 のフィールドで 7 種類のミノが出現する。
- Next キューと Hold が利用できる。
- ハードドロップで即時ロックできる。
- SRS 相当の回転と壁蹴りが動作する。
- T ミノを回転でスロットに入れ、3-corner 条件を満たしてロックすると T-Spin と判定される。
- T-Spin Double を成立させると `T-SPIN DOUBLE` が表示され、通常 Double より高いスコアになる。
- Tetris または T-Spin 系ライン消去を連続させると Back-to-Back が継続する。
- 40 Lines モードでは 40 ライン消去時にタイムが記録される。
- macOS の `.app` からゲームを起動できる。
